值得一度的文章

2018年10月3日 12:07

随着从学校走向工程，我们不断地在技术领域里扩展着足迹，也逐渐从熟悉走向陌生。从C语言到Go，从排序算法到工业算法，从SQL到数据库运维，从HTTP到TSL IPV6，从 CLI 到工业界广泛应用的 Web，从 Java Swing 到日趋成熟的移动应用，从Windows环境到Linux环境，从独立开发到协同版本控制，从手动构建部署到自动构建部署…

这个过程是漫长又迷惑的，当我们需要更可靠更专业的知识时，却发现不再有人来教我们，我们必须学会自学；当我们主动寻找一些教程时，却被一些搜索引擎误导，被五花八门的碎片文章充斥了头脑；当我们遇到问题时，去Stack Overflow暂时的解决了，却没有了解问题的原因，之后接二连三的依然出现问题，我们没有随着问题的逐渐解决变得明智，反而变得越发依赖。

我们不甘在迷惑中迷路的，自要寻找出路。寻找出路是不能忘了来路，我们的一切问题都是从工程（或研究）中来的，不可一头扎进专业名词里，忘记学习的初衷是应用。“学而不思则罔，思而不学则殆”，我们的出路就是 —— 清醒地思、有效地学、扎实地用。

凡遇到问题，多问几个为什么，去追求技术背后的原理，去分析方案背后的原因。深入理解了技术，这是我们分析和解决问题的灵感之源。这个过程当然是需要一定悟性的，也是痛苦的，但这种痛苦是面对问题的痛苦，而快乐的奖赏总在这种痛苦之后，而浮于表面总要受逃避问题的长期痛苦，再令你麻木。一个对技术有所追求的人是乐于清醒的。

思考和学习不是抠头发，要有根据。官方文档通常是很好的上手点，通常包括Overview、Guide、Tutorial、Example，看完这些就应该对一项技术是用来干什么，解决什么问题，适合什么场景，基础的用法有初步的了解，你就能决定是否要继续学习。深入地学习就需要寻找好的教材，好的文章。如果没有，就去社区和别人讨论，请教大牛，或者通过阅读源码分析，当有了自己的理解之后也能写成文章供后人参考。但切忌为了表现自己的“才识”而写文章，如果已经有更好的文章，不妨引用，莫要为虚荣而做重复的工作，把精力用到那些未开采的领域，创造性的领域。

To be the first, the best, or the different.
